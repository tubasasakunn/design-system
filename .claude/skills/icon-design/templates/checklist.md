# アイコン採用チェックリスト

新規アイコン1個を `sources/icons/<style>/<category>/<name>.svg` に追加する前に通す。
詳細な理論・出典は [../REFERENCE.md](../REFERENCE.md)、好みの境界は [DESIGN.md](../../../../DESIGN.md)。

## A. メタファー・意味（描く前に決める）
- [ ] **単一概念**を表す（1アイコン=1意味。焦点が複数なら分割）
- [ ] **resemblance（実物の描写）優先**。抽象記号を使うなら既に慣習化されたものだけ（虫眼鏡=検索/歯車=設定/ハンバーガー=メニュー）
- [ ] 文化依存の図像を主たる意味伝達に使っていない（手のジェスチャ、「チェック=正解」前提、色単独の意味）
- [ ] 命名は **描画対象**ベースの kebab-case（`floppy-disk` であって `save` ではない）。用途語は将来の alias/tag メタ層へ
- [ ] 名前は **同じ style 内で一意**（別 style の同名は意図的に可）

## B. グリッド・幾何
- [ ] `viewBox="0 0 24 24"`、グリフは**ライブエリア ~20x20**（各辺 ≥2 の余白／最低でも 1px パディング）に収める
- [ ] 支配的な形を**キーライン**に合わせた（円=直径20 / 正方形=18 / 縦長=16w×20h / 横長=20w×16h）。円は正方形より大きい＝オーバーシュート込み
- [ ] 座標・アンカーを**整数グリッド**に乗せた（小数は丸め、弧の中心もできるだけグリッドへ）
- [ ] 斜めは原則 **45°**（その他を使うなら 15° 刻み・自前規約として ADR 化）

## C. ストローク・形（style 規約）
- [ ] `stroke="currentColor"` / `fill="none"`、**固定色を埋めていない**
- [ ] `stroke-width="2"`（rounded/sharp 共通。視覚的重さを揃える）
- [ ] rounded → `linecap=round linejoin=round` ＋形状角丸 2px（8px未満の小形状は 1px）
- [ ] sharp → `linecap=butt linejoin=miter`、鋭角の bevel フォールバックを確認した
- [ ] **2px は偶数幅** → 中心線を整数座標に置きピクセル境界を対称に跨がせた（半ピクセルぼけ回避）

## D. 光学補正
- [ ] **重心で視覚的に中央寄せ**した（数学的 bbox 中心ではない）。対称形は真の中心へ
- [ ] **ブラーテスト**: `circle`/`square` と並べてぼかし、自分のアイコンだけ濃く（重く）見えない
- [ ] 円・三角・尖端は枠を少しオーバーシュートさせ、隣接アイコンと知覚サイズが揃う

## E. SVG 衛生（build 前）
- [ ] `width`/`height` 属性なし（サイズは消費側 CSS）。`viewBox` は必ず残す
- [ ] 装飾の `id`/メタデータ/エディタ名前空間/不可視ジオメトリを残していない
- [ ] 穴あき形状は `fill-rule="evenodd"` を確認（リング・くり抜き）
- [ ] アクセシビリティ属性（`role`/`aria-*`/`<title>`）は**原本に焼き込まない**（消費側の判断）

## F. ハウススタイル適合（DESIGN.md §2）
- [ ] 地は `rounded`、`sharp` 等は「意図したアクセント」として足している（無秩序な混在でない）
- [ ] ミニマル：意味が伝わる最小の線。装飾ディテールを足していない

## G. ビルド確認
- [ ] `npm run build` が通り、Web（`dist/web/icons/...`・`icons.ts`）と Swift（`Icons.xcassets`・`DSIcon.<Style>`）の両生成物が壊れていない
- [ ] `npm run build:preview` でカタログに出て、cream light/dark 両方で視認できる
