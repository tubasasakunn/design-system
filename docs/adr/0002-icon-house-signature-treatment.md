# ADR 0002: 全アイコンに一律のハウス署名トリートメントを焼き込む

## Status

Accepted — 2026-06-28

## Context

アイコンの原画は外部セット（Lucide 等）を `CLAUDE.md` 許容のもとローカル取り込みしている。カタログ `icon.json` の全 1203 種を Lucide から正規化生成した結果、見た目が **Lucide 素のまま**＝「借り物のセット」になり、このデザインシステム固有の佇まい（オリジナリティ）が無かった。

一方で、オーナーの要求は「**全てにオリジナリティを出す／ただし統一感を持たせる**」こと。両立には次の制約がある:

- **統一感** … 1203＋アクセントの全点で逸脱が無いこと。個別に手で味付けすると不揃いになる。
- **マルチPF** … Web と Swift で同一にレンダリングできること（`CLAUDE.md`）。Web 専用になる SVG フィルタ（`feTurbulence` 等の手描き風）や、Xcode が解さない表現は不可。
- **審美眼** … `DESIGN.md` の「余白多め・やわらかい線・静けさ・ミニマル」に沿うこと。
- **可読性** … 過度に細い線・小さすぎるグリフは UI で破綻する。

個別の手描き化（座標の個別摂動）は統一感・品質リスクが高く、SVG フィルタはマルチPF を壊す。

## Decision

原画の出自に関わらず、**全アイコン（全スタイル）に完全に同一の幾何トリートメント（ハウス署名）を `build` 時に機械的に焼き込む**。`DESIGN.md §4.2` に正典として記述し、内容は以下:

- **ゆったりインセット**: 中心 (12,12) を不動点に **0.875 倍**へ縮小（`<g transform="translate(1.5 1.5) scale(0.875)">` で全子要素を包む）。ライブエリア実効 ~18.4px（Lucide 素 ~22px より余白約2.5倍）。
- **やわらかい光学ストローク**: 属性 `stroke-width="2"` を保ちつつスケールにより実効 1.75px。
- スタイル差別化は終端処理のみ: `rounded`=round/round（地）、`sharp`=butt/miter（アクセント）。インセット・実効ストローク・不動点中心は共通。
- 純粋な標準 SVG（`<g transform>` ＋属性のみ、フィルタ/ラスタライズ不使用）で Web/Swift 双方同一描画。

## Consequences

- 利点: 全点が寸分違わぬ作法＝**統一感**を機械的に保証。Lucide 素と明確に異なる「空気の通った・やわらかい」佇まい＝**オリジナリティ**。`DESIGN.md` の審美眼を符号化。再生成しても逸脱しない。
- 欠点: 実効ストロークが 1.75px に下がるため、極小サイズでの線の存在感はわずかに弱まる。原画より一回り小さく描画される（密度の高い UI ではサイズ調整が要る場合がある）。
- 中立: 署名は `build`（原本 SVG に焼き込み済み。`sources/` が正本）に内在。将来パラメータ（スケール/実効ストローク）を変える場合は本 ADR を supersede する。`hand` 等の新スタイル追加時も §4.2 の署名は共有する前提。
